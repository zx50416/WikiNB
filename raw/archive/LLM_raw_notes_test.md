# LLM 筆記（Raw 測試內容）

LLM 就是 large language model，好像是大型語言模型？chatgpt claude gemini 都是

它主要就是根據前面文字猜下一個字，不是真的有腦袋/理解世界，但因為資料很多所以看起來蠻像人

token = AI讀文字的單位，英文一個字可能拆好幾個，中文不知道是不是一個字一個token。對話太長會用很多token，然後好像有context window，超過就可能忘記前面講的

prompt很重要，就是你跟AI講的話
prompt如果只打：幫我寫一篇報告
=> 回答一定很廢
應該要講 主題、對象、字數、格式、語氣
例如「寫給國中生看的500字環保報告，不要太難，要3個重點」
但其實prompt寫再好也不一定對

AI幻覺 hallucination：AI會亂掰，而且掰得很像真的
他可能會編論文、資料、網址，重要資料要自己查
AI沒有上網的話也不一定知道最新的事，訓練資料有截止日期

RAG是什麼
retrieval augmented generation
先找資料再回答？
像是公司把內部規定、pdf、文件丟進資料庫，AI先找相關段落，才用那些內容回答
這樣比直接問AI好，至少有資料來源
但如果找錯資料/資料本來就錯 還是會錯

fine tune 跟 RAG 不一樣
fine tuning是再訓練模型，讓它學某個領域，例如醫療或法律
感覺很貴，而且需要很多整理好的資料
RAG應該比較像考試可以翻參考書，fine tune像把知識直接訓練進去？
不確定這個比喻對不對

LLM可以做：
翻譯、摘要、寫信、寫程式、想點子、解釋東西
但不能完全取代人，尤其是醫療/法律/投資
因為他不會負責

今天想到：LLM不是搜尋引擎
google是找網頁，LLM是直接生成答案
但是現在很多LLM也有search所以又有點混在一起

補充：temperature好像越高，回答越有創意但也越亂
低一點比較穩定，寫程式或查資料可能要低
